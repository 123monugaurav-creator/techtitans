Why normal deletion isn't enough

When you "delete" files or empty the Recycle Bin, the operating system typically removes only references to the data — the underlying bytes remain until they are overwritten. Forensics tools can recover these remnants. On SSDs, wear‑leveling and remapped blocks make predicting where remnants live harder; cryptographic erasure or manufacturer secure‑erase commands are recommended.

Risks of incomplete deletion

  • Identity theft — lost documents (IDs, bank statements)
  • Credential leaks — saved passwords, API keys
  • Corporate non‑compliance — failing data disposal audits
  • Liability on resale or disposal of drives

How Tech Wipe Works

File Wipe

Enumerates files and performs secure overwrites (1‑pass, DoD 3‑pass, Gutmann 35‑pass, or cryptographic overwrites). After overwriting itself, the tool securely removes file records and generates hashes for verification.

Drive Wipe

Performs sector-level overwrites and can attempt SSD secure-erase procedures when supported. Optionally attempts to clear HPA/DCO regions (requires admin/low-level access).

Certificate & Audit

Captures metadata (timestamp, target, method, hashes) and signs the certificate with a key. Saves both PDF and JSON for human and machine verification.

Offline & Bootable

Generate a bootable USB or ISO for offline sanitization — ideal for secure facilities and environments where network isolation is required.
